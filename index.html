我是周姵萱，畢業於輔仁大學企管系，擁有三年工作經驗，包括兩年半的正職和半年的實習，涵蓋電商和教育服務業。 轉職的動機主要有三點：
# 對程式的興趣，過去接觸程式的經驗中，覺得寫程式很神奇，過程中充滿樂趣和成就感。
# 想要提升競爭力，學習寫程式可以為我的職涯帶來更多的選擇。
# 想創造可能性，我想透過學習寫程式，讓我有機會可以親手創造出對人類有幫助的工具。

# 學校資源：大學期間因著對程式的好奇，選修過App Inventor 2手機應用程式開發 、python 程式入門、前端網頁設計的課程。 其中印象最深刻的是前端網頁設計的課程，期末時需要小組一起將舊有網頁優化，當時要優化的網頁是就讀大學的管理評論投稿系統， 我們需要對現有的架構去改良，並將介面重新設計，需要去思考怎麼樣的畫面會對使用者最為直覺， 透過不同的角色進入到畫面也需要呈現出對應的內容與權限，其中的流程與畫面要如何串接也需要重新梳理。 我們透過HTML將內容劃分層級與切割區域，用CSS去指定顏色與風格呈現。 這堂課讓我體驗到創建網頁的樂趣，同時也獲得很大的成就感，並打從心裡覺得程式真的太神奇了，原來寫程式到實際產出的過程是這麼一回事。
期末小組實作作品參考

# 網路課程：利用閒暇時間透過線上課程自學程式基礎，主要使用的課程平台是Codecademy，目前有接觸到的內容是Python、JavaScript、Kotlin的基本編程概念。 最近接觸到彭彭的線上課程，目前已經上了前端HTML、CSS、JavaScrip的課程，接下來會安排時間往其他系列的教學課程學習。 我喜歡彭彭的講解方式，相較於其他線上平台的教學課程，彭彭教學讓我更容易理解，也可以在最後的實作環節中再次確認自己的理解是否正確。 這讓我覺得WeHelp是一個對於程式小白來說友善但又不失挑戰的地方，也更堅信想要參與WeHelp Bootcamp的意念。

# 網路資料搜尋：透過網路資料了解相關資訊與技術，查詢特定功能的實現方式，以及學習標籤的使用方法與限制。 我使用搜尋引擎進行搜索，並參考medium部落格、w3schools平台等資源累積我的知識基礎。

# 每日固定學習時間：建立每天固定的學習時段，確保一天內有多段完整的時間專注於訓練營，無論是教學內容或任務執行。

# 分階段執行：將學習內容和任務目標分為小階段，每次集中精力學習一個主題或功能。這不僅有助於理解和消化所學，也更有助於安排進度和任務追蹤。

# 制定學習計畫：列出每周的學習目標、里程碑和需完成的任務，再安排每日的進度，確保學習過程按計畫進行，並可以在每周結束時順利完成當周任務。

# 反思和調整：定期反思學習進展，加強不足的地方，並根據需要調整學習計畫，確保持續進步。

目前尚未有特定的公司選擇，未來選擇公司的主要條件是能提供具挑戰性和發展性的工作環境。 我希望能加入一個充滿學習機會且能夠不斷成長的團隊，透過參與各種專案，貢獻我的技能並累積實戰經驗。 我相信在這樣的環境中，我能夠充分發揮自己的潛力並取得更多的成就。

上一份工作中曾在非技術背景、沒有相關的經驗的情況下，接手了一個開發到1/3被暫停的的系統專案。 因為技術知識不足，加上很多東西是第一次接觸，以及上線時程壓力，內心充滿焦慮與不安，常常一個人加班到很晚，然後晚上又因憂慮而睡不好覺。 那段期間可以說是我目前經歷過最大的挑戰，不僅是挑戰我的抗壓能力也是在考驗我的應變處理能力。 我沒有太多時間被自己的情緒困綁住，我很快的找到自己焦慮不安情緒的來源，是因為我擔心自己在沒有技術背景與相關經驗的情況下無法勝任。

找到焦慮不安的源頭後，除了持續進行例行性的任務與工作外，我透過搜尋網路資料、閱讀相關書籍，幫助自己盡快充實相關的知識。 參加線上講座聆聽不同領域的專案經理分享他們的實際經驗，從不同的講座與分享中汲取養分。 除此之外我也與團隊溝通討論，找出自己在這個專案中可以貢獻的地方，藉此讓專案運作更順利。 與此同時將自己的擔心與面臨到的情況跟主管交流，並持續保有運動的習慣，適度的排解壓力與情緒。

在這次申請網頁的開發過程中，我想做一個能夠將內容隱藏再展開的功能，但遇到了一些困難，不知道這個功能叫什麼、不確定要用什麼關鍵字去搜索。 我先嘗試模糊搜尋，透過在搜尋引擎試試看輸入關於這個功能描述的關鍵字，結果在搜尋結果中找到了一些線索，讓我可以逐步縮小範圍，最終在w3schools找到了我想要的功能範例和對應的程式碼。 一開始抱持的想法是我只要跟之前在學校實作的時候一樣，把code直接複製貼上就好，這樣最保險也一定能成功運作。 但實際貼上後發現與預期不同，範例提供的預設值與我所期望的呈現相反，這讓我明白也意識到不能盲目的複製貼上，而是需要先讀懂程式碼然後根據需求的去改寫。

在這次申請網頁的開發過程中，我對於程式開發有了更深的了解，以前我認為特定的功能會對應特定的寫法， 但透過這次實作，我發現要實現相同的畫面呈現，實際上可以有多種不同的方式，不同的寫法都能達到相同的效果，主要取決於當下的限制和用途。 同時我也體悟到，想要成為工程師，除了需要良好的邏輯以外，更重要的是具備尋找並應用資料的能力， 要能明確知道自己的需求，然後精準的找到相關資料，最後能夠理解並調整程式碼以符合需求。

我覺得"誠信"是最重要的人際交往原則，在誠信的原則下對話或是溝通才有意義，這樣建立的關係才能穩固持久。
